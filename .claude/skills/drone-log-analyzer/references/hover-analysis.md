# ホバリング地点の抽出・一覧化

## 手順

1. `output/flights/` 配下の全フライトJSONを読み込む
2. 各フライトの `hover_points[]` 配列を取得
3. 全フライト分を結合して一覧化する

## 抽出フィールド

各ホバリング地点について:
- `flight_id`: フライト識別子
- `lat`, `lon`: 緯度経度（WGS84）
- `alt_m`: 高度（MSL、メートル）
- `address`: 逆ジオコーディング住所
- `start_time_utc`: ホバリング開始時刻
- `duration_sec`: 滞空時間（秒）
- `camera_yaw_deg`: カメラ方位角（度、北=0、SRTデータがある場合のみ）

## カメラピッチ角の取得

`hover_points` にはピッチ角が含まれない。必要な場合は `track[]` から取得する:

1. ホバリングの `start_time_utc` に一致する `track` ポイントを検索
2. そのポイントの `camera_pitch_deg` を取得
3. ピッチ角データがない場合は null（DJI SRTにピッチ角が含まれないケースがある）

## 集計・分析の観点

- **住所別集計**: address でグルーピングし、地域ごとのホバリング回数と合計滞空時間を算出
- **長時間滞空の抽出**: duration_sec が長い順に並べ、特に注視された地点を特定
- **高度別分布**: alt_m の分布から調査高度のパターンを把握
- **カメラ方向の傾向**: camera_yaw_deg から操縦者が注視していた方向を特定

## ホバリング判定基準

判定はコンバータが実施済み。基準の詳細は `.claude/rules/hover-detection.md` を参照:
- 速度 < 0.3 m/s が 5秒以上連続した区間
