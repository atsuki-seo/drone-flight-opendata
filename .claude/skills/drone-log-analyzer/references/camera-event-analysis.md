# カメライベントの抽出・分析

## 手順

1. `output/flights/` 配下の全フライトJSONを読み込む
2. 各フライトの `camera_events[]` 配列を取得
3. 全フライト分を結合して一覧化する

## 抽出フィールド

各カメライベントについて:
- `flight_id`: フライト識別子
- `time_utc`: イベント発生時刻
- `lat`, `lon`: 緯度経度
- `yaw_from_deg`: 変化前の方位角（度）
- `yaw_to_deg`: 変化後の方位角（度）
- `delta_deg`: 変化量（度、正=時計回り、負=反時計回り）

## パン操作と単発イベントの区別

### 連続パン操作
同一地点（ホバリング中）で短時間に複数のカメライベントが連続する場合、操縦者が360°パン撮影を行っている。

判定基準:
- 同一 `flight_id` 内で `time_utc` の差が 5秒以内
- `lat`, `lon` の差が < 0.0001度（約10m）
- `delta_deg` の符号が同一方向（一方向にパン）

例: route11-4 で 22件のカメライベントが集中 → 360°パンを複数回実施

### 単発イベント
移動中にカメラ方向を急変させた場合。操縦者が何かを発見して注目した可能性が高い。

## 集計・分析の観点

- **地点別集中度**: 緯度経度でクラスタリングし、イベントが集中する地点を特定
- **フライト別件数**: フライトごとのイベント数から注目度の高いルートを特定
- **方位変化パターン**: delta_deg の分布から全方位パン vs 特定方向注視を区別

## カメライベント判定基準

判定はコンバータが実施済み。基準の詳細は `.claude/rules/camera-events.md` を参照:
- ジンバルyawの1秒間の変化が 15度以上
- SRTデータが存在しない場合は空配列
