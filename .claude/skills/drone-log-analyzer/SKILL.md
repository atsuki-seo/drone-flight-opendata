# ドローン飛行ログ分析ワークフロー

## 最初にユーザーへ確認すること

分析を開始する前に、以下をユーザーに質問して確認する:

1. **分析対象のフライトデータ**: 変換済みJSONの場所（`output/` ディレクトリのパス、または `index.json` のパス）
2. **動画ファイルの有無と場所**: 映像抽出が必要な場合、元の動画ファイル（mp4等）がどこにあるか
3. **分析の目的・範囲**: 全ステップを実行するか、特定のステップ（例: ホバリング分析のみ）に絞るか

これらが明確になってからワークフローを進めること。

## 分析ワークフロー

以下の手順で分析を進める。各ステップの詳細は `references/` 配下のリファレンスを参照。

### Step 1: 全体概要の把握

`output/index.json` を読み込み、全フライトの概要を把握する。
ルート別のフライト数、飛行時間、ホバリング回数を集計する。

- 詳細: **`references/flight-overview.md`**

### Step 2: フライト連続性の分析

同一ルート内のフライトがDJI録画分割（約228秒）による分割かどうかを判定する。
ルート間の時間的・空間的間隔を分析する。

- 詳細: **`references/flight-continuity.md`**

### Step 3: ホバリング地点の分析

全フライトから `hover_points` を抽出し、一覧化する。
滞空時間、カメラ方向、住所別の集計を行う。

- 詳細: **`references/hover-analysis.md`**

### Step 4: カメライベントの分析

全フライトから `camera_events` を抽出し、一覧化する。
連続パン操作と単発イベントを区別する。

- 詳細: **`references/camera-event-analysis.md`**

### Step 5: 撮影範囲の推定

ホバリング地点でのカメラFOVとピッチ角から、地上の撮影範囲（footprint）を計算する。

- 詳細: **`references/footprint-estimation.md`**

### Step 6: 動画ファイルの管理（映像抽出が必要な場合）

元データから動画ファイルをリポジトリにコピーし、フライトJSONの参照を更新する。

- 詳細: **`references/video-management.md`**

### Step 7: 映像・画像の抽出（動画がある場合）

ffmpegを使用して、ホバリング地点の動画クリップ・代表フレーム画像、
カメライベント地点のフレーム画像を抽出する。

- 詳細: **`references/media-extraction.md`**

### Step 8: 抽出マニフェストの生成

抽出した映像・画像のメタデータを `extract_manifest.json` に出力する。

- 詳細: **`references/extract-manifest.md`**

## 分析パターン

1. **カバレッジ分析**: 全フライトのbboxを重ねて、調査済み/未調査エリアを特定
2. **注目地点分析**: hover_pointsとcamera_eventsを集約して、重点調査エリアを特定
3. **時系列分析**: 日時順にフライトを並べて、調査の進捗を追跡
4. **比較分析**: 石川県（被災地）と徳島県（平時）の飛行パターンの違いを分析

## データ構造の理解

- `summary` だけで1フライトの全体像を把握可能（LLMには最初にここだけ渡す）
- `track` は詳細分析が必要な場合のみ参照する（トークン効率を考慮）
- `index.json` で全フライトの一覧を俯瞰できる

## 各フィールドの分析的意味

- `hover_points` = 操縦者がその場で滞空し注視した地点 → 被害確認や要注意箇所の候補
- `camera_events` = カメラの向きが急変した地点 → 操縦者が何かを発見して注目した可能性
- `bbox` = フライトのカバー範囲 → 複数フライトの重複・空白エリアの分析に使用
- `total_distance_m` + `duration_sec` = 飛行効率の指標
